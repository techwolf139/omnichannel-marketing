## 模块设计

### 1. auth.py - 认证模块

**职责**: 管理微信公众号AccessToken

**核心方法**:
- `get_access_token()`: 获取有效Token
- `refresh_access_token()`: 刷新Token

**Token管理策略**:
- Token有效期7200秒（2小时）
- 提前300秒（5分钟）刷新
- 支持并发安全

### 2. client.py - API客户端

**职责**: 封装微信公众号API调用

**核心方法**:
- `create_draft()`: 创建图文草稿
- `update_draft()`: 更新草稿
- `publish_draft()`: 发布草稿

**错误处理**:
- 自动Token刷新
- 限流退避重试
- 异常分类处理

### 3. cover_generator.py - 封面生成器

**职责**: AI生成文章封面图

**核心方法**:
- `generate_cover(topic, style)`: 根据主题生成封面

**风格选项**:
- `default`: 简约商务风格
- `minimal`: 极简留白风格
- `modern`: 科技感渐变风格

### 4. formatter.py - 内容排版器

**职责**: Markdown转微信公众号HTML

**核心方法**:
- `format_content(markdown, theme)`: 转换并应用主题

**主题样式**:
- `default`: 经典微信风格（16px正文，#333颜色）
- `minimal`: 极简风格（18px正文，#444颜色，大留白）
- `modern`: 现代卡片式（卡片阴影，圆角边框）

## 数据流

### 发布文章流程

```
用户输入 → 内容排版 → 封面生成 → 创建草稿 → 预览确认 → 正式发布
```

### 草稿更新流程

```
获取草稿ID → 内容修改 → 重新排版 → 更新草稿
```

## 接口对接

### 微信公众号API

**基础地址**: `https://api.weixin.qq.com/cgi-bin/`

**关键接口**:

| 接口 | 方法 | 用途 |
|------|------|------|
| `/token` | GET | 获取AccessToken |
| `/draft/add` | POST | 新建草稿 |
| `/draft/update` | POST | 修改草稿 |
| `/freepublish/submit` | POST | 发布草稿 |
| `/media/uploadimg` | POST | 上传图片素材 |
