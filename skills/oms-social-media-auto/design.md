# 全平台社交媒体分发 - 详细设计

## 1. 架构设计

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      SocialDispatcher                       │
│                   (分发调度器 - 统一入口)                      │
└────────────────────┬────────────────────────────────────────┘
                     │
         ┌───────────┼───────────┐
         ▼           ▼           ▼
┌─────────────┐ ┌──────────┐ ┌──────────────┐
│ ContentTransformer │ Dispatcher Core │ Platform Registry │
│   (内容转换器)      │   (调度核心)      │   (平台注册表)      │
└─────────────┘ └──────────┘ └──────────────┘
                     │
         ┌───────────┼───────────┬────────────┐
         ▼           ▼           ▼            ▼
┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│  Zhihu   │  │  Weibo   │  │   XHS    │  │ Twitter  │
│  Adapter │  │  Adapter │  │  Adapter │  │  Adapter │
└──────────┘  └──────────┘  └──────────┘  └──────────┘
```

### 1.2 流程设计

```
内容创建 → 内容转换 → 平台分发 → 结果聚合 → 状态追踪
   │          │          │          │          │
   │          │          │          │          └─ 异步轮询
   │          │          │          └─ 统一格式返回
   │          │          └─ 各平台并行/串行
   │          └─ 按平台规则适配
   └─ 统一内容模型
```

## 2. 认证流程

### 2.1 各平台认证方式

| 平台 | 认证方式 | Token有效期 | 刷新机制 |
|------|----------|-------------|----------|
| 知乎 | OAuth 2.0 | 30天 | refresh_token |
| 微博 | OAuth 2.0 | 长期 | 自动续期 |
| 小红书 | OAuth 2.0 | 2小时 | refresh_token |
| Twitter | OAuth 1.0a / 2.0 | 长期 | 重新授权 |

### 2.2 Token管理

- 自动检测token过期
- 支持自动刷新（知乎、小红书）
- 过期时返回特定错误码，由调用方处理

## 3. 数据模型

### 3.1 SocialContent (统一内容模型)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| content_id | str | 是 | 内容唯一ID，UUID |
| title | str | 否 | 标题，长文平台使用 |
| body | str | 是 | 正文内容 |
| media | list | 否 | 媒体文件路径列表 |
| topics | list | 否 | 话题/标签列表 |
| mentions | list | 否 | @提及用户列表 |
| visibility | str | 否 | 可见性，默认public |
| scheduled_at | datetime | 否 | 定时发布时间 |
| source_url | str | 否 | 原文链接 |

### 3.2 DispatchResult (分发结果)

| 字段 | 类型 | 说明 |
|------|------|------|
| content_id | str | 内容唯一ID |
| platform | str | 平台标识 |
| status | str | 状态: success/pending/failed |
| platform_content_id | str | 平台返回的内容ID |
| platform_url | str | 平台内容链接 |
| published_at | datetime | 实际发布时间 |
| error_message | str | 错误信息 |

## 4. 平台适配规则

### 4.1 知乎适配

- **内容类型**: 文章/回答
- **标题**: 必填，5-100字
- **正文**: 支持Markdown
- **图片**: 无限制，建议单张<10MB
- **话题**: 最多10个话题

### 4.2 微博适配

- **内容类型**: 博文/图文
- **长度**: 5000字（普通用户）
- **图片**: 最多18张
- **视频**: 15分钟以内
- **话题**: #话题# 格式

### 4.3 小红书适配

- **内容类型**: 图文笔记
- **标题**: 20字以内
- **正文**: 1000字以内
- **图片**: 1-18张，3:4比例最佳
- **标签**: #标签 格式

### 4.4 Twitter适配

- **内容类型**: 推文/线程
- **长度**: 280字（普通用户）/ 4000字（Twitter Blue）
- **图片**: 最多4张
- **视频**: 512MB以内，2分20秒以内
- **话题**: #话题 格式

## 5. 模块设计

### 5.1 dispatcher.py

- **SocialDispatcher**: 主调度器类
  - `dispatch()`: 分发到指定平台
  - `dispatch_all()`: 分发到所有平台
  - `get_status()`: 查询发布状态
  - `delete_batch()`: 批量删除

### 5.2 transformer.py

- **ContentTransformer**: 内容转换器
  - `transform()`: 将统一内容转换为平台特定格式
  - `truncate()`: 按平台长度限制截断
  - `format_topics()`: 格式化话题标签

### 5.3 platforms/*.py

各平台适配器，实现统一接口:
- `post()`: 发布内容
- `delete()`: 删除内容
- `update()`: 更新内容
- `get_status()`: 查询状态

## 6. OMS 集成点

| OMS技能 | 集成方式 | 场景 |
|---------|----------|------|
| oms-xhs-integration | 订单归因后自动发笔记 | 种草内容同步 |
| oms-promotion-engine | 营销活动自动发布 | 大促活动同步 |
| oms-one-id-merge | 多平台账号归一 | 统一用户管理 |

## 7. 错误处理

| 错误码 | 含义 | 处理方式 |
|--------|------|----------|
| 1001 | 参数错误 | 检查内容格式 |
| 1003 | token无效 | 刷新token后重试 |
| 1005 | 限流 | 指数退避重试 |
| 2001 | 内容违规 | 返回具体违规原因 |
| 3001 | 平台API错误 | 记录日志，人工处理 |

## 8. 性能优化

### 8.1 并发策略

- 平台间并行：同时调用多个平台API
- 限流保护：每个平台独立限流控制
- 超时设置：默认30秒超时

### 8.2 缓存策略

- Token缓存：避免频繁刷新
- 用户ID缓存：@提及用户解析
- 内容模板缓存：提高转换效率

## 9. MVP 暂不实现

| 功能 | 原因 |
|------|------|
| 定时发布队列 | 需引入消息队列，先实现同步发布 |
| 自动回复/评论 | 需要Webhook，MVP先不做 |
| 数据分析报表 | 依赖各平台数据API，后续迭代 |
| 视频自动压缩 | 需FFmpeg支持，先要求预处理 |
| 多语言自动翻译 | 需翻译API集成，后续扩展 |
