# 跨渠道退换货协同

## Overview

支持"线上下单线下退货"和"线下购买线上退货"的跨渠道退换货。自动冲销原始订单状态，重新计算各渠道业绩归属。

## When to Use

- 申请跨渠道退换货
- 查询退货进度
- 处理仅退款
- 审核退换货申请

**触发词**: "退货"、"退换货"、"仅退款"、"跨渠道退货"、"查询退货"

## Core Pattern

### 退换货类型

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 仅退款 | 未发货取消 / 已发货仅退款 | 自动审核/人工审核 |
| 退货退款 | 退货后退款 | 逆向物流+质检 |
| 换货 | 同SKU换码/换款 | 新品出库+旧品入库 |
| 补发 | 少件/破损补发 | 直接出库 |

### 审核策略

| 金额范围 | 审核方式 | 说明 |
|----------|----------|------|
| < 50元 | 自动审核 | 秒通过 |
| 50-500元 | AI风控审核 | 羊毛党检测 |
| > 500元 | 人工审核 | 客服介入 |

### 跨渠道退货流程

```
用户申请 → 系统审核 → 物流取件/到店 → 
质检确认 → 退款/换货 → 库存回转
```

## Quick Reference

### 跨渠道场景

**场景A**: 天猫购买 → 门店退货 → 系统退款给用户，业绩冲销给天猫

**场景B**: 门店购买 → 小程序申请换码 → 门店收到退货后系统自动换单

**场景C**: A门店试穿 → B门店取货 → B门店直接换码

## Common Mistakes

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 退货期限超限 | 拒绝处理 | 签收后7天可配置 |
| 换货次数过多 | 库存混乱 | 同一订单最多2次 |
| 业绩未冲销 | 财务差错 | 退货时自动冲销 |
