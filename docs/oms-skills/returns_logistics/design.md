# 逆向物流与残次鉴定 - 设计文档

## 1. 设计目标

构建高效的逆向物流体系，实现退货的快速回收、智能质检、以及库存的精准再流转，最大限度降低退货损失。

## 2. 核心概念

### 2.1 逆向物流状态机

```
PENDING_PICKUP → IN_TRANSIT → RECEIVED → 
QUALITY_CHECK → PROCESSING → COMPLETED
```

### 2.2 质检鉴定等级

| 等级 | 条件 | 库存去向 |
|------|------|----------|
| A | 全新，吊牌/包装完整 | 原渠道可售 |
| B | 已拆封，无损坏/污渍 | 本地门店可售 |
| C | 有轻微瑕疵（可修复） | 特卖/奥莱渠道 |
| D | 严重损坏/功能故障 | 维修或报废 |

### 2.3 库存再流转

```
质检完成 → 系统自动分流:
  ├── A级 → 返回原渠道库存
  ├── B级 → 门店就地销售库存
  ├── C级 → 标记特卖渠道
  └── D级 → 生成维修工单 或 报废处理
```

## 3. 核心能力

### 3.1 物流追踪

- 接入主流快递轨迹（菜鸟/丰巢/顺丰）
- 自动签收确认（快递轨迹+重量比对）
- 异常件自动标记（包装破损/超时未到）

### 3.2 质检任务

- 支持PDA/APP扫码执行质检
- 鉴定结果实时上传
- 支持拍照留证

### 3.3 智能分流

- 根据质检结果自动分流
- 支持"就地转售"快速处理
- 异常情况自动推送人工

## 4. 能力边界

### 4.1 支持的操作

- `track_return_package`: 追踪退货物流
- `confirm_receive`: 确认收货
- `submit_quality_check`: 提交质检结果
- `process_defect`: 处理残次品
- `generate_repair_order`: 生成维修工单
- `get_recycle_report`: 获取逆向物流报表

### 4.2 限制说明

- 质检任务需在收货后24小时内完成
- 拍照留证：每个退货件至少2张
- 维修工单生成后7天内必须处理

## 5. 错误处理

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| PACKAGE_NOT_FOUND | 物流单号未匹配 | 人工核实地址 |
| QUALITY_DISPUTE | 用户对鉴定结果异议 | 转客服仲裁 |
| REPAIR_FAILED | 维修失败 | 转为报废处理 |
| RECYCLE_OVERFLOW | 库存再流转能力不足 | 暂停就地销售 |
