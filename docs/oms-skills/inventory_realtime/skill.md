# 库存水位实时映射与防超卖

## 技能概述

**技能名称**: inventory_realtime

**核心能力**: 统揽全网库存水位，实时发布可售库存，防止超卖。

## 能力描述

### 核心功能

- **多层级库存映射**: 中央大仓、区域前置仓、门店库存、在途库存统一视图
- **实时可售发布**: 向各销售前端（天猫/小程序）实时发布可售库存
- **预扣减机制**: 下单瞬间锁定库存，支付超时自动释放

### 库存层级

```
全国总仓 (RDC)
    ↓ 区域调拨
区域前置仓 (FDC)
    ↓ 铺货
线下门店 (STORE)
    ↓ 在途
在途库存 (IN_TRANSIT)
```

### 防超卖机制

**场景 - 直播间秒杀**:
- 10万用户同时抢购1000件商品
- 系统采用"预扣减 + 延迟确认"策略
- 用户下单瞬间预扣库存
- 15分钟未支付自动释放
- 保障绝对不超卖

## 调用示例

### 自然语言解析场景

**用户说**: "查一下SKU-A001在天猫渠道的可售库存"

**AI解析后调用**:
```
inventory_realtime.query_available(
  sku_id="SKU-A001",
  channel="tmall",
  location_type="all"
)
```

**返回**: 该SKU各仓的可售库存数量，以及向天猫渠道发布的可售总量。

---

**适用场景**:
- 运营监控各渠道可售情况
- 客服解答"为什么没货了"
- 大促前检查库存准备情况
