# 库存水位实时映射与防超卖 - 设计文档

## 1. 设计目标

构建全网库存实时映射系统，保障各销售渠道的可售数据准确，杜绝超卖现象，支撑高并发场景（如直播秒杀）。

## 2. 核心概念

### 2.1 库存类型

| 类型 | 代码 | 说明 |
|------|------|------|
| 总库存 | TOTAL | 仓库中所有库存 |
| 可售库存 | AVAILABLE | 减去预占/隔离后的可卖数量 |
| 预占库存 | RESERVED | 已下单未支付 |
| 隔离库存 | RINGFENCED | 渠道专属库存 |
| 在途库存 | IN_TRANSIT | 调拨中 |

### 2.2 库存计算公式

```
渠道可售 = Σ(各仓可售) - 渠道预占 - 渠道隔离

其中单仓可售 = max(0, 总库存 - 预占 - 隔离)
```

### 2.3 防超卖机制

**预扣减策略（高并发场景）**:

```
用户下单 → 预扣库存 → 返回"库存锁定" → 
  ├── 支付成功 → 确认预扣 → 库存出库
  └── 超时未支付 → 释放预扣 → 库存可售
```

**多层缓存架构**:

```
前端缓存 (30sTTL) → 实时队列 → 后端DB
    │                      │
    └── 高并发时承担读压力    └── 写请求
```

## 3. 核心能力

### 3.1 库存查询

- 支持按SKU/仓库/渠道多维度查询
- 支持查询可售/已预占/已隔离各状态库存
- 数据延迟：不超过1秒

### 3.2 库存同步

- 实时同步各仓WMS系统库存变动
- 支持全量对账和增量校正
- 异常库存自动告警

### 3.3 超卖防护

- 库存扣减原子性保证
- 乐观锁 + 重试机制
- 超卖告警实时推送

## 4. 能力边界

### 4.1 支持的操作

- `query_available`: 查询可售库存
- `query_detail`: 查询库存明细（各仓分布）
- `reserve`: 预占库存
- `confirm_reserve`: 确认预占（支付成功）
- `release_reserve`: 释放预占（超时/取消）
- `sync_inventory`: 触发库存同步

### 4.2 限制说明

- 单SKU单次预占上限：99件
- 预占超时时间：15分钟（可配置）
- 历史库存快照保留：90天

## 5. 错误处理

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| INSUFFICIENT_STOCK | 库存不足 | 返回剩余可售数量 |
| RESERVE_TIMEOUT | 预占超时 | 自动释放 |
| SYNC_LAG | 同步延迟 | 返回估算值并标记 |
| OVERSELL_ALERT | 超卖告警 | 紧急通知运营 |
| WMS_CONNECTION_ERROR | WMS连接异常 | 切换备用数据源 |
