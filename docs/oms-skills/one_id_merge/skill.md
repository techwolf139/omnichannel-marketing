# One-ID 归一与消费者画像融合

## 技能概述

**技能名称**: one_id_merge

**核心能力**: 将分散在多个渠道的消费者身份碎片合并为唯一超级ID，沉淀全渠道消费行为。

## 能力描述

### 核心功能

- **多渠道身份识别**: 通过手机号、OpenID、UnionID、会员卡号等线索关联消费者身份
- **One-ID归一算法**: 基于置信度评分自动/手动合并重复身份
- **消费者画像构建**: 整合浏览轨迹、购买历史、偏好标签形成完整单客视图

### 支持的渠道

| 渠道类型 | 身份标识 |
|----------|----------|
| 线下门店 | 手机号、会员卡、扫码OpenID |
| 天猫/京东 | 平台OpenID + 手机号绑定 |
| 抖音/快手 | 抖音OpenID、快手OpenID |
| 微信生态 | UnionID、小程序OpenID、企业微信会员ID |
| 品牌小程序 | 自主注册手机号、会员体系 |

## 调用示例

### 自然语言解析场景

**用户说**: "帮我查一下手机号138****8888的消费者，在所有渠道的购买记录"

**AI解析后调用**:
```
one_id_merge.resolve(identifier="138****8888", identifier_type="phone")
```

**返回**: 该消费者在所有渠道的合并画像，包含购买历史、偏好标签、互动轨迹。

---

**适用场景**: 
- 导购想了解顾客全渠道行为
- 客服需要跨渠道核实会员身份
- 营销系统需要精准圈人
