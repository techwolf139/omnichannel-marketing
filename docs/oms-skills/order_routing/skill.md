# 智能订单路由与拆合单

## 技能概述

**技能名称**: order_routing

**核心能力**: 基于算法决定订单最优履约路径，支持订单智能拆分与合并。

## 能力描述

### 核心功能

- **智能寻源**: 计算成本最低、时效最快的履约路径
- **订单路由**: 订单自动分配给最优仓库/门店
- **拆合单决策**: 自动评估拆单发货或合并发货的成本与时效

### 路由策略

| 策略类型 | 触发条件 | 路由结果 |
|----------|----------|----------|
| 就近履约 | 收货地址5公里内有门店有货 | 门店发货+同城骑手 |
| 成本最优 | 门店无货，系统自动计算 | 区域仓快递 |
| 时效优先 | 顾客选择极速达 | 就近门店优先 |
| 利润最优 | 高价值订单 | 优先分配利润空间大的履约方 |

### 拆合单场景

**场景A - 订单拆分**:
- 顾客购买的衣服在A仓，鞋在B仓
- 系统评估"拆成两个包裹" vs "B仓调拨A仓合并发货"
- 选择成本更低/时效更快方案

**场景B - 订单合并**:
- 同一顾客短时间内多笔订单
- 系统自动合并为一个包裹发货

## 调用示例

### 自然语言解析场景

**用户说**: "查一下订单号A123456的履约路径和当前状态"

**AI解析后调用**:
```
order_routing.get_routing_detail(order_id="A123456")
```

**返回**: 订单履约路径（从哪个仓/门店发货、快递公司、预计时效）。

---

**适用场景**:
- 客服解答顾客物流咨询
- 运营监控各仓/门店的订单分发情况
- 系统自动处理订单路由决策
