# 营销推广系统对接规范

本文档定义 OMS 系统与主流营销推广平台的对接规范，涵盖流量获取、用户互动、私域运营等核心功能。

## 1. 平台概览

| 平台 | 平台代码 | 对接类型 | 核心数据 |
|------|----------|----------|----------|
| 小红书 | XHS | 种草数据、引流追踪 | 笔记互动、用户画像 |
| 抖音 | DOUYIN | 直播带货、内容引流 | 直播间数据、短视频数据 |
| 快手 | KUAISHOU | 直播带货 | 直播间数据 |
| 微信 | WECHAT | 私域运营 | 公众号、企业微信、小程序 |

## 2. 小红书对接

### 2.1 数据类型

| 数据类型 | 获取方式 | 业务价值 |
|----------|----------|----------|
| 笔记互动数据 | 小红书开放平台API | 评估种草效果 |
| KOL合作数据 | 第三方平台/MCN对接 | 选择优质KOL |
| 引流追踪 | UTM参数 + 小程序openid | 归因分析 |
| 用户画像 | 小程序数据 | 消费偏好分析 |

### 2.2 引流归因流程

```
小红书笔记 → UTM参数追踪 → 落地页/小程序
                              ↓
                        One-ID归一 ← 手机号/OpenID
                              ↓
                        后续购买行为记录
```

### 2.3 关键指标

```
种草效果评估:
  - 笔记曝光量 (imp)
  - 笔记互动率 (点赞/收藏/评论)
  - 引流到小程序的UV
  - 引流用户的下单转化率
  - ROI = 引流带来的GMV / 投放成本
```

## 3. 抖音对接

### 3.1 数据类型

| 数据类型 | 获取方式 | 业务价值 |
|----------|----------|----------|
| 直播数据 | 抖音开放平台API | 直播效果分析 |
| 短视频数据 | 抖音开放平台API | 内容分析 |
| 粉丝画像 | 抖音企业号API | 用户洞察 |
| 引流转化 | UTM + 小程序 | 归因分析 |

### 3.2 直播订单对接

```python
# 抖音直播订单特殊性
DOUYIN_LIVE_ORDER:
  - 订单量大、并发高
  - 需要关联直播间ID
  - 秒杀/福袋活动需要防超卖
  - 直播专属优惠需要单独计算
```

### 3.3 关键指标

```
直播效果评估:
  - 直播间UV
  - 直播GMV
  - 直播间转化率 = 下单UV / 观看UV
  - 客单价
  - 退货率 (直播退货率通常较高)
```

## 4. 微信生态对接

### 4.1 对接模块

| 模块 | API | 数据内容 |
|------|-----|----------|
| 公众号 | 微信公众平台API | 粉丝数据、文章互动 |
| 企业微信 | 企业微信API | 客户数据、群发记录 |
| 小程序 | 微信支付API | 交易数据、用户OpenID |

### 4.2 私域运营对接

```
企业微信 → 客户联系记录 → OMS
                          ↓
                    导购任务下发
                          ↓
                    优惠券/活动推送
                          ↓
                    用户行为追踪
                          ↓
                    One-ID归一 ←→ 订单数据
```

### 4.3 UnionID归一

```
微信生态内归一:
  UnionID = 同一用户在微信内跨公众号/小程序的唯一标识

归一链路:
  小程序OpenID ─┐
                ├→ UnionID ─→ One-ID
  公众号OpenID ─┘
```

## 5. 跨平台用户识别

### 5.1 身份识别矩阵

| 场景 | 识别线索 | 置信度 |
|------|----------|--------|
| 同一平台多账号 | OpenID | 高 |
| 跨平台同一手机号 | 手机号 | 高 |
| 微信跨应用 | UnionID | 高 |
| 小红书→抖音 | 手机号+行为相似度 | 中 |
| 广告ID匹配 | Device ID | 低 |

### 5.2 跨平台归因

```
用户旅程:
  小红书种草 → 抖音直播下单 → 门店体验 → 小程序复购

归因模型:
  - 首次触点归因：功劳归小红书
  - 最终触点归因：功劳归小程序
  - 多触点归因：按权重分配
```

## 6. 营销数据应用

### 6.1 人群包构建

```
OMS人群包 ← 营销平台数据

标签维度:
  - 互动人群：浏览/点赞/收藏/评论
  - 引流人群：点击广告/落地页UV
  - 转化人群：下单未支付/已支付
  - 复购人群：多次购买/高价值
```

### 6.2 精准营销触发

```
触发条件 → 营销动作 → OMS执行

场景示例:
  1. 用户在小红书点赞某商品笔记
     → 推送该商品优惠券到企业微信
     
  2. 用户进入抖音直播间但未下单
     → 直播结束后推送专属券
     
  3. 用户企业微信被导购添加超过30天未互动
     → 触发唤醒任务
```

## 7. 数据安全与合规

| 合规项 | 要求 |
|--------|------|
| 用户授权 | 不得采集未授权用户数据 |
| 数据存储 | 加密存储、设置过期时间 |
| 广告追踪 | 遵守平台追踪政策 |
| 个人信息 | 不得用于非授权用途 |
