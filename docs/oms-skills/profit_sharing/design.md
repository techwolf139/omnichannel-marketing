# O2O利益分配与自动分润 - 设计文档

## 1. 设计目标

解决线上线下渠道融合带来的"业绩算谁的"这一核心利益冲突，通过灵活的规则配置和自动化分润，平衡品牌商、电商部门、加盟商、直营门店、导购等多方利益。

## 2. 核心概念

### 2.1 参与方角色

| 角色 | 代码 | 说明 |
|------|------|------|
| 品牌商 | BRAND | 品牌总部 |
| 电商部门 | ECOM | 线上运营团队 |
| 加盟商 | FRANCHISEE | 加盟门店 |
| 直营店 | DIRECT | 直营门店 |
| 导购 | GUIDE | 销售人员 |

### 2.2 分润因子

```
ProfitAllocation {
  order_id: string,
  
  goods_amount: decimal,          // 商品总额
  
  allocations: [{
    party: string,                // 参与方代码
    role: enum,                  // 角色类型
    base: enum,                  // 计算基数
    rate: decimal,              // 分润比例
    
    // 举例：门店发货佣金
    // base: "settle_price"      // 以结算价为基数
    // rate: 0.40                 // 40%
    
    amount: decimal              // 计算后的分润金额
  }],
  
  settlement_status: enum
}
```

### 2.3 分润规则优先级

| 优先级 | 规则类型 | 说明 |
|--------|----------|------|
| 1 | 导购专属码 | 有导购引流的订单优先 |
| 2 | 门店归属 | 顾客归属门店优先 |
| 3 | 发货方 | 谁发货谁拿履约佣金 |
| 4 | 默认分润 | 适用默认规则 |

### 2.4 业绩归属计算

```
场景：顾客通过A门店导购引流，在品牌小程序下单，由B直营店发货

流量归属:
  - A门店: 40% (引流贡献)
  
履约归属:
  - B直营店: 60% (发货佣金)

导购归属:
  - 引流导购: 5% (提成)
```

## 3. 核心能力

### 3.1 规则配置

- 支持按渠道/品类/门店/导购配置不同规则
- 支持阶梯式分润（如销量越高分成越高）
- 支持规则有效期管理

### 3.2 自动结算

- 订单完成后自动计算分润
- 生成结算单推送给各方
- 支持对接财务ERP

## 4. 能力边界

### 4.1 支持的操作

- `configure_rule`: 配置分润规则
- `calculate_allocation`: 计算订单分润
- `get_profit_detail`: 查询分润明细
- `generate_settlement`: 生成结算单
- `query_settlement`: 查询结算状态

### 4.2 限制说明

- 单笔订单最多5个参与方分润
- 分润比例总和不超过100%
- 历史订单分润不支持修改（需冲红重算）

## 5. 错误处理

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| RULE_NOT_FOUND | 无匹配分润规则 | 使用默认规则 |
| RATIO_EXCEED | 比例超过100% | 驳回规则配置 |
| ALLOCATION_CONFLICT | 多规则冲突 | 按优先级执行 |
| SETTLEMENT_LOCKED | 结算单已锁定 | 不允许修改 |
