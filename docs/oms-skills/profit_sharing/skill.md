# O2O利益分配与自动分润

## 技能概述

**技能名称**: profit_sharing

**核心能力**: 解决"线上卖得好，线下加盟商觉得被抢了生意"的利益冲突，实现精准的跨渠道分润。

## 能力描述

### 核心功能

- **多维度业绩归属**: 根据流量来源、导购引流、发货方等多维度分配业绩
- **自动分润结算**: 按预设规则自动计算各参与方的利润分成
- **灵活规则配置**: 支持品牌商自定义复杂的分润规则

### 分润场景

**场景A - 门店发货**:
- 天猫订单分配给加盟店发货
- 分润规则：按吊牌价40%作为发货佣金结算给门店

**场景B - 导购引流**:
- 顾客在A门店被导购添加企业微信
- 两个月后该顾客在小程序下单，由中心仓发货
- 分润规则：订单利润10%算A门店业绩，5%算导购提成

**场景C - 跨店履约**:
- 顾客在A门店试穿，B门店发货
- 分润规则：试穿门店30%，发货门店70%

## 调用示例

### 自然语言解析场景

**用户说**: "查一下订单A123456各方的分润明细"

**AI解析后调用**:
```
profit_sharing.get_profit_detail(order_id="A123456")
```

**返回**: 订单金额、各方分润比例、分润金额、结算状态。

---

**适用场景**:
- 财务核对各渠道/门店业绩
- 加盟商查询分润明细
- 导购查看个人提成
