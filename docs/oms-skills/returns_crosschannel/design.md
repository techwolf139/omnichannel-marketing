# 跨渠道退换货协同 - 设计文档

## 1. 设计目标

打破退换货的渠道壁垒，实现"Anywhere Return"——用户可在任意渠道、任意门店申请和执行退换货，同时保证订单状态、库存、业绩的精准同步。

## 2. 核心概念

### 2.1 退换货类型

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 仅退款 | 未发货取消 / 已发货仅退款 | 自动审核/人工审核 |
| 退货退款 | 退货后退款 | 逆向物流+质检 |
| 换货 | 同SKU换码/换款 | 新品出库+旧品入库 |
| 补发 | 少件/破损补发 | 直接出库 |

### 2.2 退换货流程

```
用户申请 → 系统审核 → 物流取件/到店 → 
质检确认 → 退款/换货 → 库存回转
```

### 2.3 跨渠道处理

```
OriginalOrder {
  channel: "tmall",
  store_id: null,              // 线下门店归属
  sales Attribution: {          // 业绩归属
    tmall: 70%,
    store_A: 20%,
    guide_zhang: 10%
  }
}

AfterReturn {
  original_order: "A123456",
  return_type: "refund",
  amount: 299.00,
  inventory_action: "return_to_fdc",  // 退回区域仓
  sales_reversal: {            // 业绩冲销
    tmall: -209.30,
    store_A: -59.80,
    guide_zhang: -29.90
  }
}
```

## 3. 核心能力

### 3.1 申请受理

- 支持多渠道申请入口（APP/小程序/门店POS）
- 自动识别订单来源和用户身份
- 智能推荐最近退货点

### 3.2 审核策略

| 金额范围 | 审核方式 | 说明 |
|----------|----------|------|
| < 50元 | 自动审核 | 秒通过 |
| 50-500元 | AI风控审核 | 羊毛党检测 |
| > 500元 | 人工审核 | 客服介入 |

### 3.3 库存回转

| 退货物况 | 处理方式 |
|----------|----------|
| 良品 | 就地转门店可售库存 |
| 残次 | 调拨回大仓维修 |
| 仿品/假货 | 标记销毁 |

## 4. 能力边界

### 4.1 支持的操作

- `apply_return`: 申请退换货
- `get_return_status`: 查询退货进度
- `approve_return`: 审核通过
- `reject_return`: 审核拒绝
- `track_logistics`: 追踪退货物流
- `reverse_sales`: 冲销业绩

### 4.2 限制说明

- 退货期限：签收后7天（可配置）
- 换货次数：同一订单最多2次
- 退款方式：原路返回

## 5. 错误处理

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| ORDER_NOT_ELIGIBLE | 订单不可退 | 返回不可退原因 |
| ALREADY_APPLIED | 已有进行中的申请 | 合并或拒绝 |
| RETURN_EXPIRED | 超过退货期限 | 提示转人工 |
| STOCK_NOT_AVAILABLE | 换货库存不足 | 改为退款 |
