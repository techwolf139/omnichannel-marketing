# 跨渠道退换货协同

## 技能概述

**技能名称**: returns_crosschannel

**核心能力**: 支持"线上下单线下退货"和"线下购买线上退货"的跨渠道退换货。

## 能力描述

### 核心功能

- ** Anywhere Return**: 支持跨渠道、跨门店的退换货
- **自动状态冲销**: 退换货自动更新原始订单状态
- **业绩重新计算**: 退换货后自动重新计算各渠道业绩归属

### 退换货场景

**场景A - 线上购买，线下退货**:
- 用户在天猫购买衣服
- 发现尺码不合适，前往就近门店退换
- 门店扫描订单，办理退货
- 系统自动退款给用户，业绩冲销给天猫

**场景B - 线下购买，线上退货**:
- 用户在门店购买
- 想换成其他尺码，通过小程序申请
- 门店收到退货后，系统自动换单

**场景C - 跨店换码**:
- 用户在A门店试穿，在B门店取货
- 发现尺码问题，在B门店直接换码

## 调用示例

### 自然语言解析场景

**用户说**: "帮我查一下订单A123456的退货进度"

**AI解析后调用**:
```
returns_crosschannel.get_return_status(order_id="A123456")
```

**返回**: 退货单号、退货物流、当前状态（待取件/已收到/退款中/已完成）。

---

**适用场景**:
- 客服帮用户查询退货进度
- 用户申请跨渠道退换货
- 门店接收其他渠道的退货
